Designing maikel.dev site

Maikel Thursday, 11 March 2021

Log of job done

  • Created the Paypal account for the company
  • Created the Stripe account for the company

The general idea

A website where people like Survation can log into and update their subscription status, payment method etc. But others can see the services I provide and email me questions about them.

For new clients

Remember what is value

  • Your knowledge
  • Your ability to show that knowledge into projects.

Requirements gathering. How?

Maybe use user stories to define the requirements.

This is a good research for user stories: https://www.atlassian.com/agile/project-management/user-stories

This might be handy for user stories: https://draft.io/3bq3524apuyfgspfz2cv6x2uqkv3p3h7cr68429v27tc

Definitely use this for user stories: http://app.userstory.top/analyze/

Useful API call

Using the API call below I can extract, without login in, the documents from the site that I can put in the dev site as blog posts. The kind of thing they might include are stories that I want to publish for possible users.

It'll filter by content with the dev tag.

https://www.maikel.uk/ghost/api/v3/content/posts/?key=aa0e28a02e7fd5f417e59eb3ad&filter=tag:dev

Requirements of the website

Less is more. Keep it simple.

  • Should have a list of services I offer
    To do that I should check other sites.
  • Should have a client area that allows them to manage their subscription.
  • ToS and Privacy Policy.
  • An area with documents sent to them (eg: signed contracts?) ?
  • Contact form.
  • Case Studies or Portfolio (blog format or too much hassle?)
