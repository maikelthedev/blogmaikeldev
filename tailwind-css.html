Section 2: Tailwind Fundatmentals - Part 1

What’s that “Utility First” thing

Basically you choose all the CSS in the classes but Tailwind has stuff that already builts creating things from scratch and typing CSS, I see no different between this an Bootstrap.

The difference between it and Bootstrap is the customisation.

Colours

You can use classes

bg-cyan-500
outline-cyan-500
text-red-500
border-red-500
accent-red-500
shadow-red-500

The initial word is what you want to apply the colour, the middle is the colour name, the last is from 0 (clear) to 1000 (completely black).

Spacing

You can add a file to specify what the default sizes are, but they are all already here, just learn it, it is pretty similar to Boostrap there.

Here you can target speficic size by using space-y-4 (space vertical of 4) or margin with m, padding with p.

Typography

There are 3 defaults and obviously you can use your own font, use tailwind.config.js to do that. You have 3, your sans, serif and mono but you can change those 3 to whatever you want.

Alignment is pretty easy: text-right, text-center, text-left.

Underline comes with: decoration-doble (two underlines), decoiration-dotted, even decoration-wavy.

Layout & Position

You can make fun stuff like absolute right within a parent container with “absolute bottom-0 right-0 bg-yellow-100”

intset is one interesting class. More here

Backgrounds & Shadows

You can use shadow-lg, shadow-xl, for the border. For the inside, you can use shadow-inner or even shadow-blue for cololurs.

This might make snese for the logo.

Section 3: Tailwind Fundamentals - part 2

Interactivity

You can add “hover:whatever” and describe the change or “focus:” or “active:”.

You use group when you want multiple elements to change at the same time, you add group class to the parent and to the children “group-hover:whatever” for example.

Colunns and gaps

You can put images in coluns with “column-4” (divide screen in 4 columns and) and gap you can have it or not with “gap-0” on the parent.

Flexbox

Flex creates it, flex-col puts them in columns, md:flex-row would put it in a row as you make the screen bigger.

flex-wrap drops them down when full in the flexbox.

It does make sense to add bg-blue-100 on each element while you’re tinkering with their location and size.

Grid

Use the grid class on the parent, then grid-cols-2 or whatever, and gap, depending on what you want, is similar to the way you use flexbox.

Using col-span-2 etc I can imagine this could be useful for a dashboard.

I do tend to use X more.

He repeats again and again, but do not explain why. This course is trash.

Transition

Add transition-colours with duration-1000 for 1 second. You describe the change with hover and it goes slowly from one colour (the one it has) to the one you set in hover.

Animation

You could use it for progress spinners: animate-spin

Dark Mode

If your browser or OS is in dark mode you can configure the classes to have a dark mode with “dark:”

Section 4: A better development environment

To create a tailwind project:

npm install -D tailwindcss
npx tailwindcss init // creates the tailiwind.config.js file

Create an input.css file and add the 4 directives that are needed.

All instructions are here so there’s no reason to describe them here.

Directives & Functions

You can use this to avoid repeating again and again the same classes, for example you can make all of the h1 tags to be the same so you odn’t have to redefine them on your code. Adding less classes when you code.

Section 5: TailWind CSS Mini-Projects

Order

Start with padding, then alignment, then size, then adapt to other screens, then transofrmations (hover, focus, active…etc).

It’s good practice to start mobile first.

Text-effects

Use line-through to get it scratched out and inline-block when you want it to have a background that only covers the text.
