Re-making www.maikel.dev in NextJS

Maikel Monday, 26 June 2023

I’m learning NextJS by doing this course by my favourite Udemy tutor Maximilian. 

I’m going to divide this notes in sections same as the course and I’m going to use 

Reasons I’m learning it

  • I want to design www.maikel.dev using it.
  • I want to refresh my knowledge of React since it has been a while since the last time I used it.
  • I want server-side rendering. I’ve seen how hideously slow SPA are on regularly powered computers. 
  • I think React has far more future than Angular will ever have. 
  • I ‘d like to do the Fedigrindr app using this too.

Section 1: Getting started

The simplest way to create an app is to use:

npx create-next-app

on the shell console. 

Shortcut to format document: Ctlr+Shift-i

A NextJS app seems to be a monolitic node app.

Why not choose use App Router?

Because otherwise you create a different folder structure instead of the pages.

Section 2: React Refresher

ReactJS is used (same as Angular) to make pages reactive, seem instant. Also is all about components. Webs divided in smaller parts.
