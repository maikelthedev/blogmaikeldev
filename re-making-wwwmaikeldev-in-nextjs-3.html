Re-making www.maikel.dev in NextJS

Maikel Monday, 26 June 2023 learning

I’m learning NextJS by doing this course by my favourite Udemy tutor Maximilian. 

I’m going to divide this notes in sections same as the course and I’m going to use 

Reasons I’m learning it

  • I want to design www.maikel.dev using it.
  • I want to refresh my knowledge of React since it has been a while since the last time I used it.
  • I want server-side rendering. I’ve seen how hideously slow SPA are on regularly powered computers. 
  • I think React has far more future than Angular will ever have. 
  • I ‘d like to do the Fedigrindr app using this too.

Section 1: Getting started

The simplest way to create an app is to use:

npx create-next-app

on the shell console. 

Shortcut to format document: Ctlr+Shift-i

A NextJS app seems to be a monolitic node app.

Why not choose use App Router?

Because otherwise you create a different folder structure instead of the pages.

Passing content to another component

Use props.children

<Example>

Reminder, how to send HTTP request.

Lots of packages, for this as usual. fetch() is the one built in. Example for POST request.

fetch(
    'SOMEURL', 
    {
        method: "POST",
        body: theData // Use JSON.stringify if needed
        headers: 
            {
            'Content-Type': 'application/json
            }
    }
)

You could use Redis here for simplicity while testing

Reminder on how to fetch data

fetch('someURL').then(response => {
        return response.json() // To convert it to JSON
    }).then(data => {
        doSomethingWith(data)
    });

Very probably other libraries don’t require to convert to manually to json as a step.

Using useState for loading

Use state to change what’s visible on the screen while loading. Beware a fetch should have a useEffect to avoid running for all eternity.

import {useState] from "react"

const [isLoading, setIsLoading] = useState(true);

// Then on the fetch function change it to call setIsLoading(false)

Using useEffect to avoid runnning loops

useEffect(() => {
            fetch(....)
        }, 
        [isLoading ] // Check the values on this array before executing the rest of the effect (the fetch), so in this case isLoading won't execute again. In this case isLoading shouldn't be there, and by being empty it'll only run once. 
        );

React Context

Common convention is to set up a Store folder. Alternatives to RC is Redux.

To create a context

const YourContext = createContext({
    users:  [ ],
    totalUsers: 0
});
// You also need a provider of that context
function YourContextProivider(props) {
// Here you can manage state same as in any other React component. 
    const [users, setUsers] = useState([ ]);
    
    const context = {
        users: users,
        titalUserS: users.length
    };
    
    return (
    <YourContext.Provider value={context}>
        {props.children}
    </YourContext.Provider>
    )
